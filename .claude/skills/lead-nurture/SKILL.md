# ナーチャリング5ステップメール生成

Bランク（スコア40〜69点）のリードに対して、18日間5ステップのナーチャリングメールを自動生成します。
設計思想：「売る」メールは1通も送らない。すべて「役に立つ情報」として届ける。

| Step | 送信タイミング | テーマ | 目的 |
|---|---|---|---|
| Step1 | 返信から1日後 | 同業・同規模の事例3選 | 「うちでもできる」と思わせる |
| Step2 | 4日後 | AI導入の失敗例3選 | 共感と「この人は分かってる」という信頼 |
| Step3 | 8日後 | 業種別AI活用10選 | 「これ時間かかってる」という自覚を促す |
| Step4 | 13日後 | よくある疑問・FAQ | 費用・セキュリティなどの不安を解消 |
| Step5 | 18日後 | 相談誘導（最終） | 自然な形で「30分話しませんか」へ |

## ステップ1: 対象リードの特定

引数 `$ARGUMENTS` から会社名を読み取ります。

会社名が指定されていない場合は、AskUserQuestion で会社名を確認してください。

リポジトリのルートディレクトリで以下を実行し、CRMからリードデータを取得します：

```bash
npx tsx scripts/read-crm-lead.ts --company-name "{会社名}"
```

取得したデータの `rank` フィールドを確認します：

- **Bランク**: そのまま続行
- **Aランク**: 「この企業はAランクです。Aランク対応（24時間以内のアポ打診）が推奨されます。ナーチャリングではなく、sales-outreachスキルまたは直接電話をご検討ください。」と警告し、続行するか確認
- **Cランク**: 「この企業はCランクです。3ヶ月後フォローリストに入っています。ナーチャリングを開始しますか？」と警告し、続行するか確認
- **ランク未設定**: 「この企業はまだスコアリングされていません。先にlead-scoringスキルでスコアリングを実行してください。」と案内

## ステップ2: レポートデータの読み込み

リポジトリの `data/out/` フォルダから `*_outreach-data.json` を探し、該当企業のデータを Read ツールで読み込みます。

```bash
ls data/out/*outreach-data.json
```

以下の情報をメール生成に活用します：
- `initiatives[]`: AI施策一覧（施策名・効果額・効果種別）
- `industry`: 業種
- `businessSummary`: 事業概要
- `keywords`: キーワード一覧

該当ファイルが見つからない場合は、ユーザーに以下を案内してください：
「先にgen-reportスキルでレポートを生成してください。」

## ステップ3: .envから署名情報の読み込み

リポジトリルートの `.env` ファイルから以下を読み取ります：
- `SENDER_COMPANY`: 自社名
- `SENDER_NAME`: 担当者名
- `SENDER_EMAIL`: メールアドレス
- `SENDER_PHONE`: 電話番号

## ステップ4: AskUserQuestionで生成範囲の確認

AskUserQuestion で以下を確認します：

**「全5ステップを一括生成しますか？それとも特定のステップだけ生成しますか？」**

選択肢：
- 全部（Step1〜Step5を一括生成）
- Step1のみ（同業事例3選）
- Step2のみ（AI導入の失敗例3選）
- Step3のみ（業種別AI活用10選）
- Step4のみ（よくある疑問FAQ）
- Step5のみ（相談誘導・最終）

## ステップ5: メール生成

選択されたステップに応じてメールを生成します。以下の共通ルールとステップ別テンプレートに従ってください。

### 共通ルール（鉄則）

1. **売り込みは一切しない** — 「導入」「契約」「料金」「プラン」という単語は使わない
2. **「断る許可」フレーズを毎回入れる** — 「不要であればスルーしてください」「必要なければ全く問題ありません」等
3. **相手の業種・事業内容に合わせた具体例を使う** — `businessSummary`, `industry`, `keywords` から相手に刺さる内容にする
4. **1通200〜300字以内** — 短く、読みやすく。長いメールは読まれない
5. **署名は毎回付ける** — `{担当者名}｜{自社名}\n{メールアドレス} ｜ {電話番号}`

### Step1: 同業・同規模の事例3選（返信から1日後）

```
件名：{業種}でAIを使い始めた会社の話、3つだけ共有します

{会社名}
ご担当者様

先日はご返信いただきありがとうございました。{自社名}の{担当者名}です。

{業種}の会社さんで、最近AIを取り入れた事例を3つだけ共有させてください。

1. {initiatives[0]に基づく事例：業務名+効果}
2. {initiatives[1]に基づく事例：業務名+効果}
3. {initiatives[2]に基づく事例：業務名+効果}

どれも「うちでもできそう」と思える内容かと思います。
何か気になるものがあれば、お気軽にご返信ください。

もちろん、特に今は必要ないということであれば全く問題ありません。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

事例の生成ルール：
- `initiatives[]` の施策名をそのまま使わず、「〇〇業界の会社が△△業務にAIを使って□□を実現」という事例ストーリーに変換する
- 効果は「時間が半分になった」「ミスが激減した」など定性的な表現にする（金額は出さない）
- initiativesが3つ未満の場合は、業種に合った一般的なAI活用事例で補完する

### Step2: AI導入の失敗例3選（4日後）

```
件名：AI活用で「やらないほうがいい」こと3つ、ご存知ですか？

{会社名}
ご担当者様

{自社名}の{担当者名}です。

前回は事例をお送りしましたが、今日は逆に「こうすると失敗する」というパターンを3つ共有します。

1. 目的なき導入 — 「とりあえずAI」で始めると、現場が混乱するだけで終わるケースが多いです
2. いきなり全社展開 — 最初は1部門・1業務に絞るのが鉄則です。{businessSummaryから具体的な部門/業務を1つ挙げる}あたりから始めるのが現実的です
3. 高額ツール先行 — 月額数十万のツールを入れる前に、{industry}では無料〜低コストで試せる方法がたくさんあります

失敗を知っておくだけで、いざ検討するときの判断が全然違います。

ご参考になれば幸いです。不要であればスルーしていただいて構いません。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

生成ルール：
- 失敗例2の具体的な部門/業務は `businessSummary` や `keywords` から推定する
- 失敗例3は `industry` に応じて「無料〜低コストで試せる方法」の具体例を変える

### Step3: 業種別AI活用10選（8日後）

```
件名：{業種}で使えるAI活用アイデア、10個まとめました

{会社名}
ご担当者様

{自社名}の{担当者名}です。

{業種}の会社さんで実際に効果が出ているAI活用を10個リストにしました。

1. {initiatives[0]に基づく活用例}
2. {initiatives[1]に基づく活用例}
3. {initiatives[2]に基づく活用例}
4. {業種に合った一般的なAI活用例}
5. {業種に合った一般的なAI活用例}
6. {業種に合った一般的なAI活用例}
7. {業種に合った一般的なAI活用例}
8. {業種に合った一般的なAI活用例}
9. {業種に合った一般的なAI活用例}
10. {業種に合った一般的なAI活用例}

「これ、うちでも時間かかってるな」と思うものがあれば、それがAI活用の第一歩です。

気になるものがあればお気軽にご返信ください。もちろん、今は必要なければそのままで大丈夫です。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

生成ルール：
- 1〜3は `initiatives[]` をベースに具体化する
- 4〜10は `industry` と `keywords` に基づいて、その業種で実際に使われているAI活用例を生成する
- 各項目は「業務名 — 効果（1行）」の形式で簡潔に
- 相手が「自分事」として読める内容にする

### Step4: よくある疑問FAQ（13日後）

```
件名：「AIって実際どうなの？」よく聞かれる5つの疑問にお答えします

{会社名}
ご担当者様

{自社名}の{担当者名}です。

AI活用を検討される方からよくいただく疑問を5つ、簡潔にまとめました。

Q1. 費用はどれくらいかかる？
→ {業種}であれば、月数千円〜始められるツールが増えています。大きな初期費用は不要です。

Q2. セキュリティは大丈夫？
→ 社外に出したくないデータはローカルで処理する方法もあります。段階的に進めれば安心です。

Q3. 社内の抵抗が心配…
→ 全社展開ではなく、まず1人・1業務から始めるのがコツです。成果が見えると自然に広がります。

Q4. 効果が出るまでどれくらい？
→ {businessSummaryに基づく具体的な業務}であれば、早ければ1〜2週間で変化を実感できます。

Q5. うちみたいな小規模でも使える？
→ むしろ小規模のほうが意思決定が早く、すぐ試せるのが強みです。{industry}の少人数チームでの成功例も多いです。

何か個別に気になる点があれば、いつでもご返信ください。
もちろん、今は情報収集だけという段階でも全く問題ありません。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

生成ルール：
- Q4は `businessSummary` や `initiatives[]` から具体的な業務名を入れる
- Q5は `industry` に合わせた小規模チームの事例を入れる
- 回答は全て「安心感」を与えるトーンにする。不安を煽らない

### Step5: 相談誘導・最終メール（18日後）

```
件名：{会社名}様のAI活用、30分だけ一緒に整理しませんか？

{会社名}
ご担当者様

{自社名}の{担当者名}です。
これまで何度かお役立ち情報をお送りしてきました。

もし「ちょっと気になるけど、何から手をつければいいか分からない」という状況であれば、
30分のオンライン相談で、御社に合ったAI活用の優先順位を一緒に整理できればと思います。

内容は以下のようなイメージです：
- 御社の業務の中で、AIが効きやすいポイントの洗い出し
- 最初の一歩として現実的なアクション
- 費用をかけずに試せる方法の共有

売り込みの場ではありませんので、「まだ全然決まっていない」という段階でもお気軽にどうぞ。

ご興味があれば、このメールにご返信いただくだけで大丈夫です。
もちろん、今はタイミングではないということであれば、全く問題ありません。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

生成ルール：
- 「30分」「オンライン」「無料」を明示して心理的ハードルを下げる
- 相談内容の3つの箇条書きは `initiatives[]` や `industry` に合わせてカスタマイズする
- 「売り込みの場ではない」と明示する
- これが最終メールであることを意識し、押しすぎない。あくまで「ドアを開けておく」トーン

## ステップ6: 出力

生成した各ステップのメールを、コードブロックで囲んでユーザーに提示します。

各メールは以下の形式で表示してください：

```
━━━━━━━━━━━━━━━━━━━━━━━━
Step{N}: {テーマ}（送信タイミング: {タイミング}）
━━━━━━━━━━━━━━━━━━━━━━━━

【件名】
{件名}

【本文】
{本文}
```

コピーしてそのままメールソフトに貼り付けられる形にしてください。

## ステップ7: CRM更新（任意）

AskUserQuestion で以下を確認します：

**「CRMのアクティビティに送信記録を追加しますか？」**

「はい」の場合、生成した各ステップについて、リポジトリのルートで以下を実行します：

```bash
npx tsx scripts/log-crm-activity.ts --company "{会社名}" --type "メール送信" --content "ナーチャリング Step{N}: {件名}"
```

複数ステップを一括生成した場合は、各ステップごとに上記コマンドを実行してください。

「いいえ」の場合はスキップします。

## ステップ8: ナーチャリング自動送信の登録（任意）

AskUserQuestion で以下を確認します：

**「ナーチャリングメールの自動送信を設定しますか？
（GASが送信予定日に自動的にメールを送信し、開封・クリックを追跡します）」**

「はい」の場合:

1. 担当者メールアドレスを確認します（CRMのT列から取得、なければAskUserQuestionで確認）
2. 送信開始日を確認します（デフォルト: 明日）
3. 各Stepの送信予定日を計算します（Step1=開始日, Step2=+3日, Step3=+7日, Step4=+12日, Step5=+17日）
4. 生成した5Stepのメールを以下のJSON形式で保存します:

```bash
# data/out/{会社名}_nurture.json を作成
```

JSONフォーマット:
```json
{
  "company": "会社名",
  "steps": [
    { "step": 1, "subject": "件名", "body": "本文", "scheduledDate": "YY_MM_DD" },
    { "step": 2, "subject": "件名", "body": "本文", "scheduledDate": "YY_MM_DD" },
    { "step": 3, "subject": "件名", "body": "本文", "scheduledDate": "YY_MM_DD" },
    { "step": 4, "subject": "件名", "body": "本文", "scheduledDate": "YY_MM_DD" },
    { "step": 5, "subject": "件名", "body": "本文", "scheduledDate": "YY_MM_DD" }
  ]
}
```

5. 以下を実行してCRMに登録します:

```bash
npx tsx scripts/register-nurture.ts \
  --company-name "{会社名}" \
  --email "{メールアドレス}" \
  --nurture-file "data/out/{会社名}_nurture.json"
```

「いいえ」の場合はスキップします。
