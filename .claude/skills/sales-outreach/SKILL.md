# 営業アプローチメッセージ生成

gen-reportで生成済みのレポートデータを元に、「御社専用のAI活用レポートを無料でお届けする」スタンスの初回フォームメールを生成します。

## ステップ1: 入力の確認

引数 `$ARGUMENTS` から会社名またはinputファイルパスを読み取ります。

このリポジトリのルートディレクトリ（`package.json` があるディレクトリ）の
`data/out/` フォルダから、該当する `*_outreach-data.json` ファイルを探します。

```bash
ls data/out/*outreach-data.json
```

該当ファイルが見つからない場合は、ユーザーに以下を案内してください：
「先にgen-reportスキルでレポートを生成してください。」

## ステップ2: データの読み込み

Read ツールで `data/out/{company}_outreach-data.json` を読み込みます。

JSONには以下の情報が含まれます：
- `companyName`: 会社名
- `industry`: 業種
- `businessSummary`: 事業概要（HPから抽出）
- `keywords`: キーワード一覧
- `focusAreas`: 注力部門
- `siteUrl`: ホームページURL
- `reportUrl`: 生成済みレポートURL
- `initiatives[]`: AI施策一覧（施策名・月次効果額・年次効果額・効果種別・KPI）
- `topPackage`: 最もROIの高いパッケージ情報

また、リポジトリルートの `.env` ファイルから署名情報を読み取ります：
- `SENDER_COMPANY`: 自社名
- `SENDER_NAME`: 担当者名
- `SENDER_EMAIL`: メールアドレス
- `SENDER_PHONE`: 電話番号

## ステップ3: メッセージ生成

読み込んだデータを使って、以下のルールに従いメールの件名と本文を生成してください。

### 基本スタンス

**「すでに御社専用のAI活用レポートを作成した。無料でお渡しするので、活用イメージを一緒に整理する場を30分だけいただきたい」**

売り込みではない。レポートという「情報の贈り物」を届けに来た人。
会議の目的は「AIをどう活用できるかのイメージを一緒に明確にすること」。

### 生成ルール（鉄則5か条）

1. **「レポートを届けに来た人」スタンス** — 売り込みは一切しない。導入・契約・料金という単語を使わない。すでにレポートを作ったという事実で本気度を伝える
2. **レポートの中身を具体的に見せる** — `initiatives[]` から2〜3つの施策名と効果額を列挙し、「こういう内容がレポートにまとまっています」と伝える
3. **HPの内容に具体的に触れる** — `businessSummary` と `keywords` から、相手のHPを見たことが伝わる文を作る
4. **「断る許可」フレーズを必ず入れる** — 「もちろん、今はそのタイミングでなければスルーしていただいて全く問題ありません」等
5. **アポの目的を明確にする** — 「レポートをお見せしながら、御社でAIをどう活かせるかイメージを一緒に整理できればと思います」

### メール構造（レポート提供型テンプレート）

```
件名：{会社名}様向けに、AI活用レポートを作成いたしました（無料）

{会社名}
ご担当者様

突然のご連絡失礼いたします。{自社名}の{担当者名}と申します。

御社のホームページを拝見し、
{businessSummaryから具体的に1〜2文で言及：例「広告・マーケティング領域で幅広くサービスを展開されていること」「営業力に注力されていること」}
を拝見しました。

実は、御社の事業内容をもとに
「{会社名}様がAIエージェントを活用した場合、どの業務でどれくらいの効果が見込めるか」
を独自にシミュレーションし、レポートにまとめました。

たとえばレポートの中では、

{initiatives[]から上位2〜3施策を箇条書きで紹介。例：
・{施策1の名前}（{効果種別}）→ 月{monthlyEffectYen/10000}万円相当の効果
・{施策2の名前}（{効果種別}）→ 月{monthlyEffectYen/10000}万円相当の効果
・{施策3の名前}（{効果種別}）→ 月{monthlyEffectYen/10000}万円相当の効果}

といった施策を、費用対効果やロードマップとあわせてまとめています。

決して何かを売り込みたいわけではありません。
ただ、このレポートをお見せしながら、
御社でAIをどう活かせるか、イメージを一緒に整理できればと思いご連絡しました。

もしご興味があれば、30分ほどオンラインでもご対面でもお時間をいただけませんか。
もちろん、今はそのタイミングでなければスルーしていただいて全く問題ありません。

{担当者名}｜{自社名}
{メールアドレス} ｜ {電話番号}
```

### 数字のフォーマットルール

- `monthlyEffectYen` が10000以上の場合: 「月X.X万円」（小数点以下1桁、.0の場合は省略）
- `monthlyEffectYen` が10000未満の場合: 「月X,XXX円」
- 施策名の「施策1:」「施策2:」等の番号プレフィックスは除去して自然な名前にする

### フォールバック

`businessSummary` が極端に短い（50文字未満）または「その他」のみの場合は、
HPへの具体的な言及の代わりに「御社のような{industry}の会社さんでは〜」という一般的な導入にします。

## ステップ4: 出力

生成したメールをコードブロックで囲んでユーザーに提示します。
件名と本文を分けて表示し、コピーしてすぐ使える形にしてください。

## ステップ5: CRM更新（任意）

AskUserQuestion ツールで以下を確認します：
「生成したメッセージをCRMに書き込み、ステータスを『アプローチ済み』に更新しますか？」

「はい」の場合、生成したメッセージ本文（件名は含めず本文のみ）を変数に保持し、
リポジトリのルートで以下を実行します：

```bash
npx tsx scripts/update-crm-status.ts --company-name "{会社名}" --outreach-message "{生成した本文}"
```

注意: `--outreach-message` の値にはシェルエスケープが必要です。
改行は `\n` に置換し、シングルクォートで囲んでください。
メッセージが長い場合は、一時ファイルに書き出して `cat` で渡す方法も可です：

```bash
# 一時ファイル経由の例
cat > /tmp/outreach-msg.txt << 'OUTREACH_EOF'
（生成した本文）
OUTREACH_EOF
npx tsx scripts/update-crm-status.ts --company-name "{会社名}" --outreach-message "$(cat /tmp/outreach-msg.txt)"
```

「いいえ」の場合はスキップします。
