# 製本の流れ（CSV → Excel / スプレッドシート）

提案シート型で出力した CSV を、**Excel や Google スプレッドシートで「正本」のような見た目に整える**までの流れです。

---

## 1. 出力されるもの

- **ファイル**: `data/out/{企業名}_proposal_sheet.csv`
- **中身**: 列は「ブロック種別, 施策名, 項目, 値, 単位, メモ」。  
  - ブロック種別が「共通」の行 → 共通パラメータ  
  - ブロック種別が「施策」の行 → 各施策のパラメータ（施策名でまとまる）

---

## 2. 製本のイメージ（Excel / スプレッドシート）

1. **CSV を開く**  
   - **Excel**: 「データ」→「テキストまたはCSVから」で CSV を選び、**ファイルの元の形式で「65001: Unicode (UTF-8)」を選択**してから完了。文字化けする場合はこれで読み直す。  
   - **Google スプレッドシート**: 「ファイル」→「インポート」で CSV をアップロード。文字エンコーディングは「UTF-8」のまま。

2. **正本レイアウトに整える**  
   - 1行目: タイトル「考えられる施策」など  
   - 共通ブロック: 「共通」でフィルタした行だけ見て、列を **項目 | 値 | 単位 | メモ** の形に並べる（見出し行を1行入れる）  
   - 施策ブロック: 「施策」でフィルタし、施策名ごとにセクション分け。各セクションで **項目 | 値 | 単位 | メモ** の表にする  

3. **体裁を整える**  
   フォント・罫線・列幅を調整し、クライアント提出用の「提案書」として仕上げる。

---

## 3. まとめ

| 段階 | やること |
|------|----------|
| 1. 生成 | `npm run proposal:example` で CSV を出力 |
| 2. 取り込み | Excel または Google スプレッドシートで CSV を開く |
| 3. 製本 | 共通・施策ごとにセクション分けし、正本と同じ列（項目・値・単位・メモ）でレイアウトする |
| 4. 提出 | 必要に応じて PDF 化や印刷 |

**CSV = データ**、**Excel/スプレッドシート = 製本（見た目を整えた提案書）** という役割分担です。
